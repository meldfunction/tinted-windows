Open toolkit

Already a
privacy buff?
Build with pane.

You don't have to use everything. pane's core components are independently usable. Drop the alias CLI into your own workflow, run the dashboard over your existing aliases, or use the breach monitor as a library. Take what works. Leave what doesn't. Fork it all.

Community release — these are early tools built in the open. APIs may change. Nothing here is production-guaranteed. PRs, issues, and forks are the point.
pane — use one layer or all of them
# just the alias layer import { AliasService } from 'pane/alias' const a = await alias.create({ name: 'notion' })

Modular components

Use one layer.
Use all of them.

Each component works standalone. Bring your own alias provider, your own card service, or drop the monitoring layer on top of aliases you already manage elsewhere. No lock-in.

CLI
pane alias
Alias management from your terminal
Create, list, burn, and monitor alias emails without MCP, Claude, or any AI plumbing. Works standalone with your Addy.io or SimpleLogin API key. Output is plain text or JSON for piping into your own tools.
npx pane alias create notion
pane alias list --json | jq '.[] | select(.status=="warn")'

Dashboard
pane dashboard
Alias health at a glance
Local web dashboard for all active aliases: health status, message counts, breach signals, incident log. Runs entirely local — no cloud, no telemetry, no accounts. Hit localhost:7000. Dark mode only. Bring your own aliases from any provider.
npx pane dashboard
npx pane dashboard --port 7000 --provider addy

Library
pane monitor
Continuous breach detection
Node.js library for scheduled alias inbox monitoring. Emits events on unexpected senders, volume anomalies, and probable sale signals. Drop into any Node process or trigger via cron. No daemon required. Extend the event handlers for your own notification pipeline.
import { BreachMonitor } from 'pane'
m.on('breach', ctx => webhook(ctx))

CLI
pane card
Virtual cards, zero friction
Create, fund, freeze, and expire Privacy.com virtual cards from the terminal. $0 default limit — fund only when you decide to pay. Each card is locked to a context name for auditability. US only (Privacy.com restriction).
npx pane card create figma --limit 0
npx pane card freeze shopsite-app

Library
pane context
Local SQLite envelope store
Lightweight SQLite-backed store for alias + card contexts. Saves at ~/.pane/contexts.db. Import as a library if you're building your own tooling on top of pane's data layer. No cloud, no sync, no external accounts. Fully local.
import { ContextStore } from 'pane'
const ctx = await store.get('notion')

Terminal view

If a dashboard feels like
too much UI

Everything the dashboard shows is available as plain-text terminal output. Pipe it. Script it. Put it in your tmux statusbar. pane is not opinionated about how you want to see the data.

pane — terminal
# full status overview $ pane list notion [Addy.io] clean 14d 3 msgs figma [Addy.io] clean 3d 1 msg morning-brew [Addy.io] ⚠ anomaly 47d 11 msgs from 3 domains shopsite-app [Addy.io] breach 62d 34 msgs ← pharmaadvertiser.com
# audit a specific context $ pane audit shopsite-app ⚠ breach signal detected context: shopsite-app alias: shop-z4k@pane.alias enrolled: shopsite-app.com unexpected: pharmaadvertiser.com (34 msgs since Jan 14) verdict: probable data sale event → run pane expire shopsite-app
# burn it $ pane expire shopsite-app ✓ alias shop-z4k@pane.alias deleted ✓ card ····1193 frozen ✓ context tombstoned — they can no longer reach you
# pipe JSON into your own tooling $ pane list --json | jq '.[].status' "clean" "clean" "warn" "breach"
Community

Build on it.
Improve it.

pane is open source and designed to be extended. The surveillance industry has professional tooling. The privacy community should too.

01
Data broker sweep automation
The Big-Ass Data Broker Opt-Out List documents 200+ broker opt-out procedures. Many are automatable. A pane module that submits removal requests, tracks re-acquisition, and logs non-compliance for escalation would be high-value for the community. The legal escalation pipeline from AGENTS.md is the natural next step.
open an issue →
02
Non-US virtual card integrations
Privacy.com is US-only. Revolut and Wise cover non-US users but with weaker API surface. If you've built a working card module for Revolut, Wise, or a regional equivalent, a PR adding it to src/card.js as an alternative provider would extend pane to the rest of the world.
pull request →
03
Photo cloaking pipeline
Fawkes and LowKey work but require manual per-image processing. A pane module that auto-runs Fawkes + EXIF strip on any image before it's attached to an outgoing alias email would close the photo metadata gap automatically. Python, runs locally, no cloud.
Fawkes docs →
04
Playwright enrollment — src/browser.js
The most-needed contribution. src/browser.js is currently a stub. The NEXT-tier feature — pane enroll <url> doing real stealth form fills with playwright-extra — is blocked on this file being written. The architecture is in server.js. The browser automation layer just needs to be built.
