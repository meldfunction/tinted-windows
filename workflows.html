Workflow — Claude Desktop / Anthropic
Set up pane as an MCP server in Claude
1
Clone the repo and install
required ▶

Open a terminal. Clone pane into your home folder and install dependencies:

$ git clone https://github.com/meldfunction/pane ~/pane
$ cd ~/pane
$ npm install

This installs dotenv and better-sqlite3. No telemetry. Nothing phones home.

VPN first: Before cloning or creating any accounts, connect your VPN. The IP you use when creating Addy.io and Privacy.com accounts will be logged by those services.
2
Create your Addy.io account and get an API key
required ▶

Go to addy.io and create a free account. Use a real email you control — this is your catch inbox, where all alias mail gets forwarded.

After verifying your email: go to Settings → API and click Create API key. Copy the key immediately — you won’t see it again.

Optional but worth it: Add a custom domain under Settings → Domains. Your aliases will then look like notion-x7k@yourdomain.com instead of @addy.io. Some platforms reject alias domains; a custom domain makes aliases look like normal email addresses.
3
Create your Privacy.com account and get an API key
required — US only ▶

Go to privacy.com. Create a free account and link a bank account (required — they need a funding source for virtual cards).

Go to Account → Developer and generate an API key. Copy it.

Outside the US? Skip Privacy.com for now. Use alias-only contexts. Non-US card alternatives: Revolut virtual cards, Wise. See services.html for the full list.
4
Configure .env with your API keys
required ▶
$ cd ~/pane
$ cp .env.example .env

Open .env in any text editor. Fill in at minimum:

# Your alias provider key
ADDY_API_KEY=your_key_here

# Your card provider key (skip if non-US)
PRIVACY_API_KEY=your_key_here

# Your region (affects generated fake identities)
PANE_BASE_REGION=US-East

# Default card limit: 0 = funds only when you load them
CARD_DEFAULT_LIMIT_CENTS=0

Save the file. Verify .env is in .gitignore (it is by default). Never commit this file.

5
Generate your alias seeds
required ▶

Before signing up for anything, generate random word-pair seeds. Each service you use will get its own unique seed.

$ python3 scrambler.py --count 20 --export

This writes 20 seeds to alias-seeds.txt. Try it here:

seed generator preview
Click generate to see sample seeds
Rule: One seed per service. Never reuse. The seed is not a secret — the alias email derived from it is what you protect.
6
Add pane to Claude Desktop
required ▶

Find your Claude Desktop config file:

# macOS
~/Library/Application Support/Claude/claude_desktop_config.json

# Windows
%APPDATA%\Claude\claude_desktop_config.json

Open the file (create it if it doesn’t exist) and add:

{
  "mcpServers": {
    "pane": {
      "command": "node",
      "args": ["/Users/yourname/pane/server.js"]
    }
  }
}

Replace the path with your actual path to server.js. Restart Claude Desktop. You should see a hammer icon in the chat interface.

7
Test it: sign up for something
test ▶

In Claude Desktop, try this:

you: "Sign me up for Morning Brew newsletter"

Claude calls: pane_create_alias("frost-ridge")
✓ frost-ridge-b3c@yourdomain.com created

you: "What is that alias email?"
Claude: Your alias for Morning Brew is frost-ridge-b3c@yourdomain.com

you: "Has anything arrived there?"
Claude calls: pane_check_inbox("frost-ridge")
✓ 1 message — welcome from morningbrew.com

Workflow — Free trial backstop
$0 card means they can’t charge you when the trial ends
1
Pick a seed for this service
required▶

Every service gets its own seed. Never reuse.

pick a seed
Click generate
2
Create a full envelope (alias + $0 card)
required▶
$ pane create-context amber-circuit
✓ amber-circuit-4a1@yourdomain.com
✓ card ····4242 · $0 limit

The card has a $0 limit. Any charge attempt fails. The merchant cannot charge you when the trial ends — or ever — unless you fund it.

3
Sign up with alias email and virtual card
required▶

Go to the service’s signup page. Use:

  • Email: your alias (e.g. amber-circuit-4a1@yourdomain.com)
  • Card: the virtual card from pane (run pane list to see card details)
  • Name: whatever pane generated — or leave as-is

The service has your alias, a card it cannot charge, and a generated name. Nothing links back to you.

4
When the trial ends: decide
decision▶

Option A: You want to keep it. Fund the card for the subscription amount:

$ pane card fund amber-circuit --amount 1200
✓ card funded: $12.00

Option B: You don’t want it. The card is already at $0. No charge is possible. You’re done. Optionally burn everything:

$ pane expire amber-circuit
✓ alias burned · card frozen

Workflow — Newsletter / canary
One alias per newsletter. When they sell it, you’ll know.
1
Build your newsletter alias map
required▶

Every publication gets its own alias. This is your canary network — each alias is a named accusation waiting to be triggered.

# Create all at once
$ pane create-alias frost-ridge
$ pane create-alias cedar-vault
$ pane create-alias willow-sluice
$ pane create-alias copper-root
2
Sign up with each alias
required▶

Go to each newsletter’s signup page. Enter the corresponding alias email. No card needed for free newsletters.

Keep your alias-map.md updated as you go. This is your audit trail.

3
Check inbox and watch for anomalies
ongoing▶
$ pane check-inbox frost-ridge
5 messages — all from morningbrew.com — clean

# Three weeks later...
$ pane check-inbox frost-ridge
⚠ 1 unexpected message from partnerads.net

$ pane audit frost-ridge
⚠ breach signal: partnerads.net
verdict: Morning Brew sold your alias
4
Burn when breached
if needed▶
$ pane expire frost-ridge
✓ alias frost-ridge-b3c@yourdomain.com deleted
✓ Morning Brew can no longer reach you
✓ incident logged: morningbrew.com → sale confirmed

Create a new alias if you still want the newsletter. Morning Brew gets one more chance under a new seed.

Workflow — Placeholder account
Book under a placeholder identity. Update your real name by phone at the last mile.
1
Create a placeholder context
required▶
$ python3 scrambler.py --count 1
birch-portal

$ pane create-context birch-portal
✓ birch-portal-7f3@yourdomain.com
✓ card ····9182 · $0 — placeholder, unfunded
2
Book online using placeholder identity
required▶

Go to the airline or hotel site. Complete the booking using:

  • Name: use pane’s generated name (e.g. Jordan Ellis) — or any placeholder
  • Email: birch-portal-7f3@yourdomain.com
  • Card: ····9182 at $0 — the booking hold will fail or you’ll need to fund it slightly for the hold
Some booking sites require a card charge upfront. Fund the card to exactly the hold amount before completing the booking. You’re still protected — the merchant can’t charge more than the limit.
3
Retrieve your confirmation number
required▶
$ pane check-inbox birch-portal
1 message — confirmation from aa.com
ref: ABC123

Note your confirmation reference. You’ll need it for the phone call.

4
Call to update your real name (the last mile)
required for travel▶

Call the airline or hotel directly and say:

"Hi, I have a reservation under Jordan Ellis, confirmation ABC123.
I need to update the name to [your legal name] for travel ID purposes."

They update the record. Your legal name is in their reservation system. Your email, IP, device fingerprint, and browsing history are not — those belong to the placeholder identity.

Why phone? A phone call creates no browser fingerprint, no cookie, no IP address logged to your account. The call center record is siloed. It does not get aggregated into a cross-platform advertising profile.
5
Fund the card and travel
required▶
$ pane card fund birch-portal --amount 34500
✓ card funded: $345.00

After the trip, expire the context. The booking platform keeps a profile of Jordan Ellis — not you.

$ pane expire birch-portal
✓ alias burned · card frozen

Workflow — CLI / standalone
Full alias management from your terminal. No AI required.
1
Install and configure (same as Claude Desktop steps 1–4)
required▶

Follow steps 1–4 of the Claude Desktop workflow. The install and config are identical — the difference is you talk to the CLI instead of to Claude.

2
Core CLI reference
reference▶
# Create alias only
$ pane create-alias frost-ridge

# Create card only
$ pane create-card frost-ridge --limit 0

# Create both (envelope)
$ pane create-context amber-circuit

# See everything
$ pane list

# Read alias mail
$ pane check-inbox amber-circuit

# Audit for breach signals
$ pane audit amber-circuit

# Burn alias + freeze card
$ pane expire amber-circuit

# JSON output (pipe to jq etc.)
$ pane list --json | jq '.[] | select(.status=="warn")'

Workflow — Journalist / high-risk
Your device sends a Signal message. The agent does the rest.
1
Read AGENTS.md before starting
required▶
Read first: AGENTS.md contains the full security model, jurisdiction table, E2E coverage map, and hosting checklist. This workflow assumes you have read it.

This workflow is for journalists, activists, and anyone whose physical safety depends on separating their device from their account management. It is not for casual use.

2
Set up an agent server in a privacy-respecting jurisdiction
required▶

Provision a VPS in Iceland (1984 Hosting, Flokinet) or Switzerland (Infomaniak). Pay with Monero if possible. SSH key auth only.

# On the agent server
$ git clone https://github.com/meldfunction/pane /opt/pane
$ openssl rand -hex 32
8f3a9c2d... ← your AGENT_SECRET, store it

$ echo "AGENT_SECRET=8f3a9c2d..." > /opt/pane/.env
$ echo "ADDY_API_KEY=your_key" >> /opt/pane/.env
$ chmod 600 /opt/pane/.env

$ python3 /opt/pane/pane-agent.py monitor &
3
Set up Signal bot or .onion command channel
required▶

The command channel is how you talk to your agent without exposing your IP or device. Two options:

Signal bot (simpler): Install signald on the agent server. Link it to a Signal account (ideally a burner number). You message that number from your personal Signal to issue commands.

.onion endpoint (stronger): Run a Tor hidden service on port 8080. Add to /etc/tor/torrc:

HiddenServiceDir /var/lib/tor/pane/
HiddenServicePort 80 127.0.0.1:8080

Your .onion address is in /var/lib/tor/pane/hostname. Access it from Tor Browser on your device.

4
Filing workflow: device → Signal → agent → platform
operation▶
Your phone (Signal message): "publish this draft to [publication platform]" ↓ E2E encrypted — Signal protocol Agent server (Iceland): Receives intent Looks up clean alias identity for this publication Spins ephemeral browser context Routes through Tor exit node Logs into publication platform under alias Submits draft Burns browser context ↓ Confirmation only Your phone: "published — ref #XY23" Your device never touched the publication platform.
